# TypeScript / JS Playbook

## When To Use
- Frontend components
- Node services
- Utility libraries

## Acceptance Criteria
- Types defined (no implicit any)
- Public APIs documented
- Edge cases handled
- Unit tests added

## Rules
- Strict typing enabled
- No console logs in production code
- No unused exports
- Prefer functional components

## Verification
- tsc passes
- Tests pass
- Lint passes
